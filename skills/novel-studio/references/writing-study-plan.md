# 写作学习计划（Novel Studio）

本文件用于把“学习写作”从泛泛积累，收束为**可落地、可复盘、可反打回项目正文**的训练流程。

原则：**学任何技巧，都要回到正在写的小说里验证。**

---

## 一、适用场景

当用户提出以下需求时，进入本训练参考：

- 想系统学习小说写作 / 网文写作
- 想知道当前最该补什么写作能力
- 想把学习内容整合进正在写的项目
- 想做“拆书 + 反打到自己作品”的训练
- 想建立长期写作训练计划

如果用户当前重点是直接写章、补设定、审稿，可优先走主技能流程；仅在用户明确提出“学习 / 训练 / 提升写作能力”时使用本参考。

---

## 二、核心原则

### 1. 学习目标不是懂概念，而是让作品变好看

学习结果必须能回答至少一个问题：

- 人物有没有更立得住？
- 章节有没有更有追更感？
- 设定有没有更自然地落进情节？
- 感情线有没有更真、更疼、更不油？
- 高潮有没有不只是揭谜，而是改变关系和局势？

### 2. 先补短板，不追大而全

优先补当前作品最限制上限的能力，而不是随机学“写作知识点”。

### 3. 学完立刻回项目验证

固定流程：

1. 明确本轮训练主题
2. 总结训练要点
3. 选定正在写的项目/章节/角色
4. 反打到项目里做一次应用
5. 复盘：哪些有效，哪些还只是理论

### 4. 用“项目驱动学习”，不要“学习驱动拖延”

不要拿“我还没学够”当成不写正文的借口。训练是为创作服务，不是替代创作。

---

## 三、推荐训练模式

Novel Studio 在以下模式之外，额外支持：

### 8. 学习模式（训练模式）

用于：
- 制定写作学习计划
- 诊断当前作品最该补的能力
- 建立分阶段训练任务
- 做“技巧 → 项目应用 → 复盘”闭环

**默认规则：**
- 学习模式不直接等同于写正文。
- 输出应尽量聚焦 1-3 个当前最关键的训练主题。
- 每个训练主题都要附带一个项目内应用任务。

---

## 四、写作学习的四阶段路线

### 阶段一：人物驱动训练

目标：让人物不是按设定行动，而是带着欲望、顾虑、伤口和误解互相拉扯。

重点学习：
- 角色此刻最想要什么
- 角色最怕什么
- 嘴上说的和心里要的如何错位
- 冲突来自需求对撞，不是单纯吵架
- 关系变化如何靠事件而不是宣告完成

常用训练任务：
- 角色三表：欲望 / 恐惧 / 底线（可读取：`references/character-three-table-template.md`）
- 角色前期动机变化表（可读取：`references/motivation-change-template.md`）
- 关系分阶段设计
- 一场“双方都不坏，但就是撞”的冲突戏设计

---

### 阶段二：章节结构训练

目标：让每一章都真正推进，而不是只有氛围和信息。

重点学习：
- 一章的目标、阻力、代价、变化
- 中途变数如何打断原本路径
- 章末为什么让人想继续看
- 情绪章、过渡章、信息章如何避免发虚

常用训练任务：
- 章节推进检查表：目标 / 变数 / 升级 / 钩子（可读取：`references/chapter-progression-checklist.md`）
- 章节删减测试：删掉这一章后，后文是否受影响
- 同一章尾换 3 种钩子类型练习

---

### 阶段三：信息控制训练

目标：让设定长在事件里，而不是像讲义。

重点学习：
- 什么信息必须现在给
- 什么信息只给半句
- 什么信息用角色误解代替直讲
- 如何通过物件、动作、反应传达信息

常用训练任务：
- 一条设定链的分章投放表
- “直讲版”改写成“场景显露版”
- 同一信息用对白 / 物件 / 反应 3 种方式表达

---

### 阶段四：追读力与高潮训练

目标：保证章节有追更感，高潮不是只揭谜，而是掀动人物关系与局势。

重点学习：
- 四类章尾钩子：情绪 / 信息 / 危险 / 选择
- 高潮的组成：真相 + 关系变化 + 局势变化 + 后续选择变化
- 爽点如何有铺垫、有见证、有代价

常用训练任务：
- 章尾钩子库
- 三个阶段高潮设计：前中后
- 把“信息高潮”改成“关系+局势双高潮”

---

## 五、项目内应用法（必须）

每轮学习必须配一组“反打任务”，优先从以下四类中选：

1. **角色反打**：给关键角色补三表、动机变化、关系推进
2. **章节反打**：重做章节推进检查表
3. **设定反打**：把一条设定链拆成分章投放方案
4. **高潮反打**：把当前高潮从“揭信息”升级到“掀关系与局势”

推荐话术：
- “先别铺太开，先练这一个”
- “这个技巧回到你这本书里，最适合打在哪一章？”
- “学完要落回项目，不然只是懂了，不是会了”

---

## 六、最常见的学习误区

### 1. 把“知道术语”误当成“会写”

知道“人物弧光、三幕、节奏、钩子”不等于会用。必须回正文验证。

### 2. 只看别人怎么写，不拆自己为什么没立住

真正的训练应该优先服务当前项目痛点。

### 3. 总在补设定，不补人物和章节推进

设定资源再强，如果人物不碰、章节不推，作品还是不耐追。

### 4. 学习变成拖延写作

如果已经知道当前最该练什么，就不要继续兜圈看理论，直接做应用任务。

---

## 七、训练输出模板

当用户请求“给我做学习计划 / 看我该学什么 / 做训练路径”时，优先按以下格式输出：

### 1. 当前问题诊断
- 最该补的 1-3 个能力
- 为什么这几个最关键

### 2. 本轮训练目标
- 本轮只练什么
- 不练什么

### 3. 训练要点
- 3-5 个最实用原则

### 4. 项目内应用任务
- 直接点名要落到哪本书、哪条线、哪几章

### 5. 复盘标准
- 做完以后，怎么判断有没有学到

---

## 八、推荐与 Novel Studio 主流程的衔接

- 用户说“帮我学习写作” → 进入学习模式
- 用户说“那把这个应用到我的小说里” → 切回设定 / 规划 / 章节 / 连载模式
- 用户说“检查我有没有学到位” → 进入审校模式

一句话：

> **学习模式负责找到该练什么；主流程负责把它真正写进作品里。**
