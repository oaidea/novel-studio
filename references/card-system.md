# Card System

> 用于统一小说项目中的人物、空间、场景对象化管理。

## 一、为什么要卡片化
长篇项目最容易出问题的不是不会写，而是：
- 信息散
- 记忆漂
- 修改后不回写
- 对象变了却没记录为什么变

卡片系统的目标是：
- 当前状态有地方看
- 变化过程有地方追
- 新信息有地方归档

## 二、对象范围

### 1. 人物
- 人物卡：当前定位、关系、边界、写法原则
- 人物变化记录：章节推进中的认知、立场、关系、状态变化

### 2. 空间
- 空间卡：固定空间结构、物件、动线、机位
- 空间变化记录：空间结构、使用状态、物件状态变化

### 3. 场景
- 场景卡：场景定位、参与人物、场景功能、动作顺序、信息释放原则
- 场景变化记录：为什么这样切、为什么这样改、影响了什么

## 三、使用原则
- 卡片本体写“现在是什么”
- 变化记录写“怎么变成现在的”
- 一类对象一套模板
- 一个对象尽量一张卡 + 一份变化记录
