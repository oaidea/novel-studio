# Event Card System

> 事件卡用于补齐“人物 / 空间 / 场景 / 时间线”之外的另一类关键对象：事件本身。

## 一、什么时候需要事件卡
当以下内容需要反复被引用，但又不适合只放在章节正文里时，建立事件卡：
- 一次火起
- 一次会面
- 一次旧物出现
- 一次身份揭露
- 一次断裂升级

## 二、事件卡记录什么
- 事件定义
- 事件参与者
- 触发原因
- 事件经过摘要
- 事件结果
- 当前状态（已结束 / 仍在发酵 / 已埋伏笔待回收）
- 关联人物 / 空间 / 场景 / 时间线

## 三、事件变化记录记录什么
- 事件在哪一章被推进
- 状态如何变化
- 影响了哪些对象卡
- 后续是否需要继续追踪
