# Packet-First Chapter Workflow

> 目标：把新章节创作标准化为“先预测依赖对象，再确定本章目的，再生成 chapter packet，再写正文”，尽量避免直接回读前文全文。

---

## 一、总流程

创作新章节时，默认按以下顺序走：

1. **基于上一章摘要，而不是上一章全文，预测本章会调用哪些对象**
2. **确认本章真正目的**（事件 / 人物 / 空间 / 时间 / 伏笔）
3. **生成本章框架**
4. **生成 chapter packet**
5. **同步更新相关卡片的预期变化**
6. **再进入正文生成**

也就是说：

> **先做对象预测与章节目标建模，再写正文。**

---

## 二、Step 1：基于上一章摘要预测本章依赖对象

新章节开始前，不默认读取上一章全文。

默认输入应是：
- 上一章结构化摘要
- 当前时间线锚点
- 当前人物状态摘要
- 当前未回收伏笔子集

然后先预测本章大概率会涉及哪些对象：

### 1. 人物
- 上一章结尾还在场的人
- 上一章刚被提起、但未正面出现的人
- 因上一章事件而必然被牵动的人

### 2. 空间
- 上一章结尾所在空间
- 本章承接动作最自然的空间
- 因事件升级而必须切入的新空间

### 3. 时间
- 上一章结束时刻
- 本章与上一章是否连续
- 是否有显式时间跳跃

### 4. 事件
- 上一章已经发生但尚未收口的事件
- 本章必须推进的事件
- 因上一章结果而新生成的事件

### 5. 伏笔
- 上一章刚埋下的伏笔
- 本章必须强化 / 回收 / 延后标记的伏笔

---

## 三、Step 2：确认本章真正目的

本章目的不能只写成“继续剧情”。

必须明确回答：

### 1. 事件目的
- 本章要推进哪件事？
- 这件事要推进到什么状态？

### 2. 人物目的
- 本章要推动谁发生变化？
- 是认知变化、关系变化、立场变化，还是行动变化？

### 3. 空间目的
- 本章是否要坐实一个空间？
- 是否要让某个空间状态发生变化？

### 4. 时间目的
- 本章时间是否要推进？
- 是否要建立新的时间锚点？

### 5. 结构目的
- 本章是承接章、推进章、揭示章、缓冲章，还是转折章？

一句话：

> **本章目的要拆成人物 / 事件 / 空间 / 时间 / 结构五层，而不是只写情节概括。**

---

## 四、Step 3：确定章节框架

在目的明确之后，再生成本章框架。

框架至少要包含：
- 开场承接点
- 中段推进点
- 本章最关键变化点
- 章末收束 / 钩子点

并且要同时标明：
- 哪个人物卡会被调用
- 哪个事件卡会被推进
- 哪个空间 / 场景卡会被调用
- 哪个时间锚点会变化

---

## 五、Step 4：生成 chapter packet

当本章框架确定后，输出 chapter packet。

packet 至少应包含：
- 本章目标
- 本章阻力
- 本章代价
- 本章结束状态
- 本章依赖的对象卡清单
- 上一章最小承接摘要
- 本章禁止事项
- 章末钩子类型

这一步之后，正文生成原则上只依赖：
- packet
- 相关卡片
- 上一章摘要

而不再默认依赖上一章全文。

---

## 六、Step 5：提前定义“本章将引起哪些卡片变化”

在写正文前，先把预计变化列出来。

### 1. 人物卡 / 人物变化记录
- 谁会变？
- 变在哪？
- 变化是否要写入 change log？

### 2. 时间卡
- 是否产生新时间锚点？
- 是否需要更新 timeline？

### 3. 事件卡 / 事件变化记录
- 本章推进了哪个事件？
- 事件状态是加深、转向，还是暂时冻结？

### 4. 空间卡 / 场景卡
- 空间是否发生状态变化？
- 场景是否形成新的稳定写法？

### 5. 伏笔记录
- 哪条伏笔被推进 / 强化 / 延后？

这一步的意义是：

> **正文不是写完才回头补结构化记录，而是在写之前就先知道这章会改动哪些卡。**

---

## 七、标准读取顺序（写新章节时）

默认读取顺序：

1. `chapter packet`
2. 上一章结构化摘要
3. 当前人物状态摘要 / 人物卡 / 人物变化记录
4. 当前事件卡 / 事件变化记录
5. 当前空间卡 / 空间变化记录
6. 当前场景卡 / 场景变化记录
7. 当前时间线锚点
8. 本章必须处理的伏笔子集

默认不读：
- 上一章全文
- 前几章全文

只有以下情况才回读全文：
- 需要精确复用对白语气
- 需要核验细微动作连续性
- 需要确认某段原文措辞
- 摘要与卡片不足以覆盖创作需求

---

## 八、写完后的回写顺序

1. 更新本章摘要
2. 更新人物变化记录
3. 更新事件变化记录
4. 更新空间 / 场景变化记录
5. 更新时间线锚点
6. 更新伏笔状态
7. 如有必要，回写人物卡 / 空间卡 / 场景卡本体

---

## 九、一句话工作原则

> **新章节先预测对象，再定本章目的，再定框架，再生成 packet，再写正文，再回写卡片。**
