# Input Pack Strategy

> 目标：将单章输入包拆成多条轨道，分别服务模型与人工。

## 一、模型输入版（极简）
- 目标：极致节省 token
- 默认只带：summary / packet / style overlay
- 适合对象变化少、章节承接简单的情况

## 二、模型输入版（标准）
- 目标：在低 token 前提下保留对象状态层
- 默认带：summary / packet / style overlay / object state summary
- 适合对象变化更多、承接要求更高的情况

## 三、人工审阅版
- 目标：给人快速核对本章准备状态
- 除核心输入外，增加 active indexes
- 适合人在正式开写前做快速检查

## 四、为什么要多档
同一个输入包同时满足模型和人工，往往会两边都不够好；
同一个模型输入包同时覆盖简单章和复杂章，也会不够精细。
所以分成：
- 模型极简版
- 模型标准版
- 人工审阅版
