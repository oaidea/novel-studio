# 信息释放控制 / Information Release Control

> 用途：管理章节和场景中的设定、真相、线索、伏笔、解释量，防止“讲太早、讲太多、讲太直、讲完就没悬念”。

---

## 一句话定义

**信息释放控制**：决定本场/本章让读者知道什么、不知道什么、误以为什么，以及这些信息通过什么载体进入正文。

核心原则：

> 信息不是作者倒出来的，是被人物行动、冲突、发现、失败、代价一点点挤出来的。

---

## 信息分层

### 1. 读者必须知道

不说就看不懂当前动作的信息。

### 2. 读者可以知道

知道后会增强理解，但不影响当前阅读。

### 3. 读者暂时不能知道

提前知道会破坏悬念、反转、恐惧或后续推进。

### 4. 读者可以误解

允许读者和人物一起误判，用于后续修正认知。

### 5. 作者自己知道即可

属于后台设定，不该进入正文。

---

## 信息载体优先级

优先顺序：

1. 行动后果
2. 具体物证
3. 人物反应
4. 对话试探
5. 记录 / 档案 / 消息
6. 简短旁白补缝
7. 大段解释（慎用）

能用前五种，就不要直接上第六、第七种。

---

## 释放节奏

### 先结果，后原因

先让读者看到异常或后果，再补必要原因。

### 先局部，后系统

先出现一个点不对，再逐渐意识到规则层出了问题。

### 先疑问，后答案

每个答案最好带出一个更大的问题。

### 先误判，后修正

让人物基于有限信息做出合理误判，再用新证据修正。

---

## 单章信息预算

一章不要同时完成太多信息任务。

建议每章控制：

- 核心新信息：1 个
- 辅助线索：2-3 个
- 误导或烟雾：0-1 个
- 暂缓解释：明确标记，不在正文里硬讲

如果一章里既解释世界观、又揭身份、又讲能力规则、又回收伏笔，通常会失焦。

---

## 解释腔危险信号

- “也就是说”后面接长段设定
- “原来……”连续解释多个因果
- 对话双方都在说彼此本来知道的事
- 角色突然像老师讲课
- 作者替角色总结意义
- 一个异常刚出现，机制立刻讲透
- 伏笔还没形成好奇心，就急着答案兑现

---

## 信息释放链路

### 悬疑链

```text
异常结果 → 局部证据 → 错误解释 → 新证据推翻 → 暂时答案 → 更大疑问
```

### 设定链

```text
规则后果 → 人物吃亏/受益 → 最小规则说明 → 新限制 → 后续应用
```

### 伏笔链

```text
不起眼细节 → 当章功能 → 延迟标记 → 后续重复变形 → 回收解释
```

### 身份链

```text
行为违和 → 第三方反应 → 旧线索重组 → 身份部分揭开 → 新风险出现
```

---

## 和其他模块的关系

- **镜头跟随**：信息只能通过当前跟随对象可感知的东西进入。
- **第三方描述**：信息可以通过旁人误读、传闻、组织评估释放，但要遵守第三方信息边界。
- **场景张力**：信息释放最好发生在阻力和代价中，而不是静态说明。
- **伏笔管理**：本章释放的信息必须标注“已揭 / 延后 / 误导 / 待回收”。

---

## 审校清单

- 本章核心新信息是什么？是否超过 1 个？
- 哪些信息是读者必须知道？哪些只是作者想讲？
- 是否提前讲透了机制或真相？
- 信息是否通过行动、物证、反应、对话进入？
- 是否存在角色互相讲废话式设定？
- 每个答案是否带出新问题或新代价？
- 是否保留了暂时不能知道的信息？
- 是否有误导项？误导是否公平？
- 伏笔释放状态是否需要回写？

---

## 输出建议

用于写作前，生成信息释放任务书：

```md
## 信息释放任务书

- 本章核心新信息：
- 必须知道：
- 可以知道：
- 暂时不能知道：
- 可以误解：
- 后台设定不入正文：
- 信息载体：行动 / 物证 / 反应 / 对话 / 档案 / 旁白
- 本章答案带出的新问题：
- 需要回写的伏笔状态：已揭 / 延后 / 误导 / 待回收
```

用于审校时，输出：

1. 必修：提前泄底、解释腔、信息越权
2. 建议：载体替换、拆分释放、误导设计
3. 可选：伏笔回写、下章疑问承接
