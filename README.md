# novel-studio

`novel-studio` 是一个统一小说创作技能，面向**长篇小说、网文连载、结构化章节写作、审稿与润色**场景。
它把原本分散的几类能力整合成一个入口：立项、设定、大纲、单章写作、连载管理、章节审校、文风优化，都可以走同一个技能完成。

当前版本：**v0.1.0**

---

## 适用场景

适合以下需求：

- 新建小说项目
- 搭建世界观、角色、力量体系
- 规划故事主线、分卷大纲、章节大纲
- 续写某一章、重写某一章
- 管理长篇连载中的状态、节奏、伏笔
- 审核章节逻辑、人设、时间线、钩子
- 润色文本，降低 AI 痕迹，让文字更自然

不适合的场景：

- 普通短文案生成
- 非小说类专业写作
- 依赖外部 API 或复杂自动化脚本的写作流水线

---

## 核心能力

### 1. 项目初始化
支持建立小说项目骨架，包括：
- 故事主线
- 世界观设定
- 角色档案
- 力量体系
- 分卷大纲
- 章节大纲
- 时间线
- 伏笔与回收
- 状态追踪
- 当前进度

### 2. 章节级结构化写作
内置章节工作流：

**预写分析 → 任务书 → 初稿 → 自检 → 润色 → 元数据更新**

适合：
- 单章创作
- 卡文时拆解本章任务
- 降低长篇写作中的断层和失控

### 3. 长篇连载管理
内置简化的三线节奏控制：
- Quest（主线推进）
- Fire（关系/感情推进）
- Constellation（世界观扩展）

用于避免：
- 主线太久不动
- 感情线长期失踪
- 世界观一直只提不落地

### 4. 章节审核
提供章节审核模板，支持检查：
- 主线推进
- 冲突强度
- 时间线
- 场景与行动逻辑
- 设定一致性
- 人设稳定性
- 钩子强度
- 节奏债务
- AI 味 / 解释腔

### 5. 文风优化
集成去 AI 味思路，重点处理：
- 解释感太重
- 句式过于工整
- 人物说话没有区分度
- 抽象词过多、动作细节过少
- 结论句太多、情绪不落地

## 设计目标

`novel-studio` 不是为了替代所有写作方法，而是想解决几个高频问题：

- 写着写着忘设定
- 章节能写出来，但结构虚
- 长篇连载越写越散
- 审稿时没有统一抓手
- 润色时只会“改得更像 AI”

所以它的定位不是“自动写神作”，而是：

**给小说创作一个统一、可复用、可迭代的工作台。**

## 当前状态

这是一个 **v0.1.0 测试版**。
已经可以实际使用，但还没有经过大量不同题材、不同项目的长期回测。

适合：
- 小范围试用
- 自己项目内先跑
- 边用边修

不建议：
- 直接当作完全成熟的终版能力描述
- 不经测试就宣称适配所有写作流派
